문제 해결 및 개선 경험

개인별 리포트 분석 결과 정적 파일 생성
서버 증설 없이 달성한 데이터 불변성 확보
문제
분석 결과 생성 후에도 제품의 변경이 분석 결과에 영향을 주지 않도록 분석 결과를 정적파일로 다룰 필요성이 있었습니다. 고객은 분석 결과를 분리되고 독립된 내용이라 이해했습니다. 분석이 추가되거나 분석 방법이 변경되었을 때 분석 결과 조회 페이지에 즉시 반영되는 구조였기에 개선이 필요했습니다. 분석이 완료된 결과는 제품 변경에 영향을 받지 않아야 한다는 요구사항을 만족시켜야 했습니다.
해결 과정
정적파일 생성 서버를 추가하려는 방향을 생각하다 팀원들과 여러 논의를 거쳐 FE의 vike 패키지를 활용하여 기존 코드 기반에서 정적 빌드가 가능한 것을 확인했습니다. 새로운 서버를 구성하지 않고 비동기 worker를 추가하여 빌드만 담당하도록 코드를 추가했습니다. 서브프로세스를 호출하여 정적파일 생성하고 S3에 업로드하는 방향으로 정적 파일 생성 작업을 진행했습니다. 정적파일 생성 결과를 한번에 업로드하기 위해 AWS S3 CLI를 사용했으며, 이 또한 서브프로세스를 호출하는 방식으로 진행했습니다. S3는 민감정보를 보관하는 비공개 버킷과 assets을 보관하는 공개 버킷을 구분했고, 공개 버킷에 CloudFront와 Route 53 레코드를 추가로 설정하여 외부 접근에 대한 캐시를 구성했습니다.
평가
부득이하게 ECS에 공통으로 사용되는 백엔드 도커 이미지 파일이 커졌으며 배포 시간도 길어졌습니다. 이는 이후 도커 이미지를 분리하는 방향으로 개선하려 합니다. 분석 요청 트래픽이 많지 않아 빌드 서버를 분리하지 않는 판단을 내렸으며, 오버스펙을 우려했습니다. 제품 사용 규모에 맞는 구조로 설계하여 요구사항을 만족했으며 제품 개선이 이전보다 자유로워졌습니다.
