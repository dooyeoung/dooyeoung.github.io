고객의 문제를 깊이 공감하고, 과감한 실행력으로 아이디어를 실제 제품으로 만들어내는 백엔드 엔지니어입니다. 레몬베이스에서 최대 30,000개 목표 조회 속도를 30s에서 0.5s로 83% 이상 단축하고, AWS 인프라 비용을 연간 5,600만 원 절감하는 등 측정 가능한 성과를 통해 비즈니스 성장에 기여해왔습니다.

특히 개인별 리포트 분석 제품을 2개월 만에 성공적으로 출시하여 고객에게 새로운 가치를 제공했습니다. 기존 R 언어로 구현된 분석 로직을 Python으로 완전히 재구현하고 강약점,자기객관화 분석부터 단일 선택, 복수 선택, 주관식 분석까지 제품 범위를 대폭 확장했습니다. 분석 결과 검증을 위한 내부 도구까지 개발하여 QA 과정의 효율성을 크게 향상시켰습니다.

저의 가장 큰 강점은 "하면 됩니다"라는 긍정적인 태도와, 문제를 마주했을 때 끈질기게 파고들어 해결하는 집요함입니다. 아이디어가 떠오르면 주저하지 않고 PoC를 통해 빠르게 프로토타입을 만들고, 동료들과 함께 더 나은 방향으로 발전시켜 나갑니다. 이러한 실행력으로 동료들로부터 "팀의 사기와 추진력을 높인다"는 평가를 받았습니다.

기술적으로는 Python, Django, AWS ECS, Terraform 등을 활용하여 확장 가능한 고품질 SaaS 제품을 개발하고 운영한 경험이 있습니다. 또한, 목표 현황 대시보드, 구성원 대량 생성 도구, 분석 검증 툴 등 다양한 내부 도구를 개발하여 동료들의 생산성을 높이는 데에도 기여하고 있습니다.

새로운 비즈니스 도메인을 깊이 탐구하는 데 큰 흥미를 느낍니다. 최근 미용사-피부 국가자격증을 취득하는 과정에서, 에스테틱 고객 재방문율, 계절별 수요 등 현업의 비즈니스를 직접 경험하고 분석하며 개발자로서 디지털 전환의 가능성과 필요성을 파악할 수 있었습니다. 이처럼 도메인에 깊이 몰입하여 사용자와 비즈니스를 이해하는 경험은, 기술로 실제 문제를 해결하는 데 가장 중요한 자양분이 된다고 생각합니다.

모든 문제를 기술로 해결하기보다 외부 서비스 도입과 운영 정책을 개선을 동료들과 적극적으로 논의하여 문제 해결에 합리적인 방법을 찾으려 합니다. 투명하고 따뜻한 소통을 통해 동료들과 함께 시너지를 내는 것을 중요하게 생각합니다. 함께 일하기 좋은 동료이자, 문제 해결에 기여하는 개발자가 되기 위해 끊임없이 배우고 실행하고 있습니다.

문제
분석 결과 생성 후에도 제품의 변경이 분석 결과에 영향을 주지 않도록 분석 결과를 정적파일로 다룰 필요성이 있었습니다. 고객은 분석 결과를 분리되고 독립된 내용이라 이해했습니다. 분석이 추가되거나 분석 방법이 변경되었을 때 분석 결과 조회 페이지에 즉시 반영되는 구조였기에 개선이 필요했습니다. 분석이 완료된 결과는 제품 변경에 영향을 받지 않아야 한다는 요구사항을 만족시켜야 했습니다.
해결 과정
정적파일 생성 서버를 추가하려는 방향을 생각하다 팀원들과 여러 논의를 거쳐 FE의 vike 패키지를 활용하여 기존 코드 기반에서 정적 빌드가 가능한 것을 확인했습니다. 새로운 서버를 구성하지 않고 비동기 worker를 추가하여 빌드만 담당하도록 코드를 추가했습니다. 서브프로세스를 호출하여 정적파일 생성하고 S3에 업로드하는 방향으로 정적 파일 생성 작업을 진행했습니다. 정적파일 생성 결과를 한번에 업로드하기 위해 AWS S3 CLI를 사용했으며, 이 또한 서브프로세스를 호출하는 방식으로 진행했습니다. S3는 민감정보를 보관하는 비공개 버킷과 assets을 보관하는 공개 버킷을 구분했고, 공개 버킷에 CloudFront와 Route 53 레코드를 추가로 설정하여 외부 접근에 대한 캐시를 구성했습니다.
평가
부득이하게 ECS에 공통으로 사용되는 백엔드 도커 이미지 파일이 커졌으며 배포 시간도 길어졌습니다. 이는 이후 도커 이미지를 분리하는 방향으로 개선하려 합니다. 분석 요청 트래픽이 많지 않아 빌드 서버를 분리하지 않는 판단을 내렸으며, 오버스펙을 우려했습니다. 제품 사용 규모에 맞는 구조로 설계하여 요구사항을 만족했으며 제품 개선이 이전보다 자유로워졌습니다.

해결 과정

SQLAlchemy-ImageAttach 패키지를 사용하여 매장 이미지를 S3에서 관리하고 있었으며 s3의 업로드된 url을 db에 저장하는 방식이 아닌 이미지 파일의 메타정보만 db에 저장하고 이미지 url 접근시 메타정보를 s3에 전달하여 url을 확인하여 전달하는 방식이였습니다.

평가
이미지 로딩 속도를 200ms 이하로 단축할 수 있었습니다. SQLAlchemy-ImageAttach 사용 장점을 유지하고 URL 조회의 단점을 구조 변경 없이 해결하였습니다.
